20 ## Build instructions
21 
22 **Inicpp** is build as shared and static library using `cmake`. There are two options to control which libraries are built, by default both are set to ON. Basic syntax is:
23 
24 ```
25 cmake [-G generator] [-DBUILD_STATIC=ON|OFF] [-DBUILD_SHARED=ON|OFF] source_dir
26 ```
27 
28 Also C++ compiler with at least C++14 support is required.
29 
30 ### Linux
31 
32 To build the library run following commands from root of source code directory:
33 
34 ```{.sh}
35 $ mkdir -p build
36 $ cd build
37 $ cmake ..
38 $ make
39 ```
40 
41 To build and run unit tests using Google Test framework run additional commands:
42 
43 ```{.sh}
44 $ make -f tests/Makefile
45 $ ./tests/run_tests
46 ```
47 
48 **Note:** For building tests you must have all the sources including git submodules. This could be done using `git clone --recursive https://github.com/SemaiCZE/inicpp.git` command when clonning or `git submodule update --init` when you already have the sources.

54 ## Examples
55 
56 ### Basic usage without schema
57 
58 ```{.cpp}
59 using namespace inicpp;
60 
61 // Load config file from file
62 config example_conf = parser::load_file("conf.ini");
63 
64 // Iterate through sections
65 for(auto &sect : example_conf) {
66  std::cout << "Section: " << sect.get_name() << std::endl;
67  // Iterate through options in a section
68  for(auto &opt : sect) {
69  std::cout << "Option: " << opt.get_name() << " with value "
70  << opt.get<string_t>() << std::endl;
71  }
72 }
73 
74 // Get known item from known section
75 try {
76  signed_t port = config["github"]["port"].get<signed_t>();
77 } catch (bad_cast_exception) {
78  // item cannot be casted to signed_t type
79 }
80 // Make reference to the section
81 section &github_sect = config["github"];
82 // Get a string always work
83 string_t port_str = github_sect["port"].get<string_t>();
84 
85 // Change some values
86 github_sect["port"].set<signed_t>(42222);
87 // Following two lines are equivalent
88 github_sect["address"].set<string_t>("www.github.com");
89 github_sect["address"] = "www.github.com";
90 
91 // Change sigle value to list by inserting an item
92 option &addr_opt = github_sect["address"];
93 addr_opt.add_to_list<string_t>("https://github.io");
94 assert(addr_opt.is_list()); // now it's a list
95 // Remove item from the list (specifying position)
96 addr_opt.remove_from_list(0); // remove first item
97 assert(!addr_opt.is_list());
98 
99 // Save changes
100 std::ofstream ofs("conf_new.ini");
101 ofs << config;
102 ```
103 
104 Without schema all items are treated as string type, but runtime conversion could be done to one of supported types if possible.
105 
106 ### Schema validation support
107 
108 Schema validation can bring you ability to make sure, that a config file contain what you expect with type safety. In addition, this implies better performance on heavy reading of validated configuration.
109 
110 ```{.cpp}
111 using namespace inicpp;
112 
113 // First prepare the schema:
114 schema conf_schm;
115 
116 // Add first section
117 conf_schm.add_section("github"); // default is mandatory
118 // Add option named "port" into "github" section. The option is mandatory
119 // and it's type is unsigned. Default value makes no sense for mandatory
120 // options. This option has no validating function, so all values are valid.
121 conf_schm.add_option("github", option_schema_params<unsigned_t> {"port", true});
122 // Mandatory string option "address" with default value "www.github.com" (one value, not list).
123 // Option is valid, if given string is greater than "aaa" (using lambda of functor class
124 // you can specify whatever you want!
125 option_schema_params<string_t> args = {"address", false, false,
126  "www.github.com", [](const string_t &str){ return str > "aaa"; }};
127 conf_schm.add_option("github", args); // template argument is deduced
128 
129 // Alternatively, create the classes first and add them to schema
130 option_schema_params<boolean_t> autoconnect_params = {"autoconnect", false, false, "off"};
131 option_schema autoconnect_option(autoconnect_params);
132 section_schema connect_section("connect", false); // optional section
133 connect_section.add_option(autoconnect_option);
134 conf_schm.add_section(connect_section);
135 
136 // Then load file with validation (from stream conf_istream, just to show a different way)
137 try {
138  config example_conf = parser::load(conf_istream, conf_schm, schema_mode::strict);
139 } catch (invalid_config_exception) {
140  // config is not valid
141 }
142 
143 // Or you can validate already loaded config!
144 config example_conf = parser::load_file("conf.ini");
145 if(example_conf.validate(conf_schm, schema_mode::relaxed)) {
146  // now the config is valid and all values are internaly
147  // stored as their correct type
148 }
149 
150 // Default configuration (schema) can be written to a file using output stream or parser save method
151 parser::save(conf_schm, "default_conf.ini");
152 ```
